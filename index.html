04 · CORE CONCEPTS
5 Pillars of
Agentic AI
01 LLM Reasoning Classify, extract, plan, generate. Think: function(input) → structured output.
02 Tools & Actions Search, call APIs, read DBs, send emails. Language becomes real-world side effects.
03 Memory Short-term (context), long-term (vector DB), episodic (history). State that persists.
04 Orchestration The router: decides which agent or tool runs next. LLM-driven or code-driven.
05 Evaluation Agents fail silently. Ship scoring, human checkpoints, and guardrails first.
05 · Real Example

"Build a customer support tool that handles refund queries"

✗ Vibe Coding
1Prompt ChatGPT → paste code into editor
2Build a chat UI with an input box
3Call LLM API on submit
4Display response in a text area
5Ship it. Celebrate.

Result: A chat wrapper. No DB, no ticket system, no state. It hallucinates order IDs.

vs
✓ Agentic Approach
1Design task graph: classify → retrieve → respond
2lookup_order(id) → real DB call
3check_policy(type) → refund rules engine
4create_ticket(data) → Jira/Zendesk
5Agent decides steps. Human approves edge cases.

Result: Classifies intent, fetches real data, applies policy, creates tickets, autonomously.

06 · How to Design
Agentic
Wireframes.

Think in flows, not screens. Before opening your IDE, answer these 5 questions.

That diagram IS your agentic wireframe.

Q1
What is the trigger?
User message · Scheduled event · Webhook · API call
Q2
What does the agent need to KNOW?
Data sources, DB, APIs, prior context
Q3
What DECISIONS does the LLM make?
Intent classification, routing, ranking, extraction
Q4
What ACTIONS can it take?
Tools: write, call API, read DB, notify, create ticket
Q5
When does a HUMAN step in?
Approval gates · Low confidence · Error recovery
07 · UNLOCK LLM POWER
LLMs are more than a chat box.
Classifier

Route tickets, tag content, detect intent, moderate text at scale. Input: raw text. Output: category.

Extractor

Pull structured data from messy PDFs, emails, and forms. Input: unstructured blob. Output: clean JSON.

Planner

Break a goal into an ordered task graph. Input: objective. Output: step-by-step execution plan.

Ranker

Score and rank candidates or search results against criteria. Input: list. Output: ranked list with scores.

Validator

Check output quality, catch hallucinations, verify claims. Input: output + ground truth. Output: pass/fail.

Stop calling it "AI".

It's a programmable reasoning engine.

10 · Avoid These
Mistakes that
kill AI products early.
01 No evals Ship a scoring system before shipping a feature. Blind deployment = silent failure.
02 Infinite loops Add max_iterations and timeout. Agents without limits loop forever on ambiguous tasks.
03 Hallucination in actions Ground all decisions in retrieved data. Never let the LLM invent IDs, amounts, or names.
04 No human in the loop Define confidence thresholds. Low confidence → escalate. Non-negotiable in fintech.
05 One big prompt Decompose into sub-agents. Each agent does ONE thing well. Compose them into a pipeline.

Most AI projects fail not because of bad models, but because of bad architecture.
